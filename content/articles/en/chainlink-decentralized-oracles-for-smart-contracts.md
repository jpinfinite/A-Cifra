[Chainlink](/artigo/chainlink-oraculos-descentralizados-smart-contracts "Chainlink: Oráculos Descentralizados para Smart Contracts"): Decentralized Oracles for Smart Contracts

The [Chainlink](/artigo/chainlink-orculos-descentralizados-para-contratos-inteligentes "Chainlink: Oráculos Descentralizados para Contratos Inteligentes") is the most widely used decentralized oracle infrastructure in the blockchain ecosystem, connecting smart contracts to real-world data in a secure and reliable manner.

## The Oracle Problem

Smart contracts are programs that execute automatically when conditions are met. However, there is a fundamental problem:

### The Connectivity Problem
- Blockchains are isolated environments
- They cannot access external data directly
- They need "oracles" to bring information

### Risks of Centralized Oracles
- **Single Point of Failure**: If the oracle fails, the contract fails
- **Manipulation**: Data can be maliciously altered
- **Censorship**: The oracle can be shut down

## The Chainlink Solution

Chainlink resolves these problems with a decentralized network of oracles:

### How It Works

**1. Data Request**
- Smart contract requests data
- Specifies parameters and sources

**2. Oracle Selection**
- Multiple independent nodes are selected
- Based on reputation and stake

**3. Data Aggregation**
- Each node searches for data independently
- Responses are aggregated
- Outlier values are discarded

**4. Delivery to Contract**
- Aggregated data is sent to the smart contract
- Transaction is recorded on the blockchain

## The LINK Token

LINK is the native token of Chainlink with essential functions:

### Payment for Services
- Users pay in LINK for data
- Node operators receive LINK

### Staking
- Node operators stake LINK
- Guarantees honest behavior
- Penalizes incorrect data

### Governance
- Participation in protocol decisions
- Updates and improvements
